1

Determinan Kinerja Sistem Bus

Kinerja sebuah sistem bus tidak ditentukan oleh satu variabel tunggal, melainkan merupakan resultan dari berbagai parameter teknis yang saling berinteraksi. Berikut adalah analisis komprehensif mengenai faktor-faktor yang memengaruhi kinerja sistem bus:

1. Lebar Bus (Bus Width)

Ini adalah faktor paling fundamental. Lebar bus data (misalnya 32-bit vs 64-bit) menentukan jumlah bit yang dapat ditransmisikan secara paralel dalam satu siklus clock. Semakin lebar bus, semakin besar bandwidth teoritisnya. Demikian pula, lebar bus alamat menentukan kapasitas maksimum memori yang dapat diakses (addressable memory space).

2. Kecepatan Clock (Clock Speed)

Frekuensi operasional bus (dalam MHz atau GHz) berbanding lurus dengan throughput data. Namun, peningkatan frekuensi fisik memiliki batasan karena masalah interferensi sinyal dan panas. Oleh karena itu, arsitektur modern sering menggunakan teknik seperti Double Data Rate (DDR) untuk mentransfer data pada kedua sisi sinyal clock.

3. Metode Sinkronisasi

Synchronous Bus: Menggunakan clock global. Cepat dan efisien untuk interkoneksi jarak dekat, namun kaku karena semua perangkat harus mengikuti kecepatan clock master.

Asynchronous Bus: Menggunakan protokol handshaking. Lebih fleksibel menangani perangkat dengan kecepatan beragam, namun memiliki latensi lebih tinggi akibat overhead sinyal kontrol.

4. Arbitrasi & Tipe Transfer

Metode arbitrasi (sentralisasi vs terdistribusi) memengaruhi efisiensi waktu tunggu (latency) saat banyak modul berebut akses bus. Selain itu, dukungan terhadap Burst Transfer (blok transfer) memungkinkan pengiriman data beruntun tanpa perlu mengirimkan alamat untuk setiap paket, yang secara drastis mengurangi overhead protokol.

2

Optimalisasi Kinerja Melalui Direct Memory Access (DMA)

Pada sistem konvensional (Programmed I/O atau Interrupt-Driven I/O), CPU bertindak sebagai perantara aktif untuk setiap perpindahan data antara memori dan perangkat I/O. Hal ini menciptakan bottleneck yang signifikan karena siklus instruksi CPU terbuang hanya untuk tugas "administratif" pemindahan data.

Teknologi Direct Memory Access (DMA) hadir sebagai solusi arsitektural untuk mendelegasikan tugas transfer data tersebut. Berikut mekanisme peningkatan kinerjanya:

  • 1. Reduksi Overhead CPU (CPU Offloading)

    CPU hanya perlu menginisialisasi proses transfer dengan mengirimkan parameter (alamat awal, jumlah blok data, dan sinyal kontrol) ke modul DMA. Setelah itu, CPU bebas untuk melanjutkan eksekusi instruksi lain (seperti kalkulasi aritmatika atau logika) secara paralel sementara DMA menangani transfer data fisik.

  • 2. Efisiensi Bus Melalui Burst Mode

    DMA dapat mengambil alih bus dalam mode burst untuk mentransfer blok data besar secepat kemampuan memori, meminimalkan jeda antar-paket yang biasanya terjadi jika CPU yang melakukan eksekusi instruksi `MOVE` berulang-ulang.

  • 3. Minimalisasi Interupsi

    Alih-alih menginterupsi CPU setiap kali satu byte atau word siap ditransfer, DMA hanya mengirimkan sinyal interupsi satu kali di akhir proses transfer blok data yang besar. Ini mengurangi context switching pada sistem operasi secara drastis.

3

Evolusi Arsitektur Bus: Tradisional vs Kinerja Tinggi

Perkembangan perangkat I/O dengan bandwidth tinggi (seperti Gigabit Ethernet dan kartu grafis modern) menuntut evolusi struktur interkoneksi bus.

A. Arsitektur Bus Jamak Tradisional

Dalam arsitektur ini, terdapat ketergantungan linier yang tidak efisien. Expansion Bus terhubung langsung ke System Bus melalui interface tunggal. Masalah utamanya adalah latensi dan bottleneck. Perangkat I/O yang cepat dipaksa beroperasi mengikuti ritme Expansion Bus yang lambat, atau membebani System Bus secara berlebihan sehingga mengganggu kinerja CPU.

B. Arsitektur Bus Jamak Kinerja Tinggi

Arsitektur modern mengatasi limitasi di atas dengan memperkenalkan konsep Mezzanine Bus (High-Speed Bus). Arsitektur ini memisahkan perangkat I/O berdasarkan kebutuhan bandwidth-nya.

  • High-Speed Bus (PCI/PCIe): Dihubungkan erat dengan sistem bus melalui bridge kecepatan tinggi. Dikhususkan untuk perangkat intensif data seperti Video Controller, Graphics, dan High-Speed LAN.
  • Expansion Bus (Legacy): Terpisah dari jalur cepat, menangani perangkat lambat seperti Serial Port, Fax, dan Modem, sehingga tidak "menyeret" turun kinerja sistem secara keseluruhan.
